Основные понятия: передача данных и рекурсия

Этот урок посвящён двум важным понятиям, с которыми связан весь курс:

1) сравнению и передаче переменных по значению и по ссылке;

2) рекурсии.

Передача и сравнение данных в JavaScript

Передача и сравнение данных в JavaScript бывает двух типов: по значению и по ссылке.

По значению передаются примитивные типы данных: строки, числа, були, символы, null и undefined.

let a = 5; // число - примитивный тип
let b = a;

a = 10;

console.log(a, b); // 10, 5

Число 5 — примитивный тип данных. Когда мы записываем значение одной переменной в другую (b = a), движок JavaScript:

1) создаёт новую область памяти,

2) записывает в переменную b ссылку на эту область.

Выделение памяти при работе с примитивными типами

В итоге получаются две области памяти и две ссылки — по одной на каждую. Так работает передача данных по значению.

По ссылке передаются не примитивные типы: объекты, массивы и функции.

let a = { value: 5 }; // объект - не примитивный тип
let b = a;

b.value = 10;

console.log(a, b); // { value: 10 }, { value: 10 }

Объект { value: 5 } — не примитивный тип данных. Когда мы записываем не примитивное значение одной переменной в другую (b = a), движок JS создаёт ещё одну ссылку на область памяти, в которой хранится этот объект. Обе переменные ссылаются на одну область памяти. При редактировании мы просто изменяем эту область памяти, а ссылка остаётся той же.

Выделение памяти при работе с не примитивными типами

То же происходит и при сравнении. Примитивные типы a и b ссылаются на разные области памяти, но хранят одно и то же значение. Поскольку это примитивный тип данных, сравнивается содержимое этих областей памяти, а оно одинаковое: 

const a = 5;
const b = 5;

console.log(a === b); // true

Сравнение примитивных типов данных происходит по значению

a и b также ссылаются на разные области памяти и содержат одинаковые значения. Но это не примитивные типы, поэтому сравниваются указатели на область памяти, а не сами значения:

const a = { value: 5 };
const b = { value: 5 };

console.log(a === b); // false

Сравнение не примитивных типов данных происходит по ссылке

В стандарте ECMA-262, который отвечает за спецификацию языка JavaScript, нет указаний как именно должны храниться данные в памяти. Поэтому фактическая реализация работы с памятью может отличаться от приведённой. Но принцип передачи и сравнения типов по значению и ссылке остаётся неизменным.

Рекурсия

Чтобы понять рекурсию, сначала нужно понять рекурсию.

Рекурсия — это вызов функцией самой себя:

function recursion() {
   console.log('recursion');
   recursion(); // функция вызывает саму себя
}

recursion();

Но не запускайте этот код — ваш браузер зависнет. При вызове функция recursion начнёт вызывать сама себя бесконечное число раз, что приведёт к переполнению памяти.

Чтобы такого не случилось, в рекурсивных функциях всегда нужно условие выхода, при котором функция не будет вызывать себя ещё раз, а сделает что-то другое. Например, вернёт какое-то значение или просто выйдет:

function recursion(n) {
   if (n === 0) {
		console.log('recursion end'); 
		return; 
	}

   console.log(`recursion ${n}`);
   recursion(n - 1);
}

recursion(4);

Теперь мы рекурсивно вызовем функцию ровно 5 раз и по условию равенства нулю выйдем. Важно обработать все случаи. Например, при передаче отрицательного числа, текущий вариант кода уйдёт в бесконечную рекурсию. Подумайте, как это исправить.

Рекурсия помогает разбить большую задачу на части и решать каждую отдельно, применяя одну функцию, а затем просто объединить результаты. Представьте, вам нужно перетасовать колоду из 100 карт. Делать это одной не очень удобно. Но вы можете раздать 4-м друзьям по 25 карт, чтобы каждый по очереди перетасовал свою маленькую колоду. Затем вы просто совместите эти колоды в одну.
