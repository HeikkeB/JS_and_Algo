Как работает setTimeout

Метод setTimeout — это по сути таймер. Вы говорите ему, что и через какой период времени нужно выполнить, и он делает это, но ровно один раз. Примеры использования:

setTimeout(() => {
	console.log('Я выполнюсь через 2 секунды')
}, 2000);

// Или так
setTimeout(console.log, 2000, 'Я выполнюсь через 2 секунды');

Ранее рассказывали, как setTimeout обрабатывается стеком вызовов и очередью событий. В этом уроке подробнее остановимся на проблеме этого метода. Примером послужит такой код:

function hardWork() {
	const start = Date.now();
	while (Date.now() - start < 3000) {}
	console.log('hard work');
}

setTimeout(function cb() {
	console.log('after callback');
}, 1000);

hardWork();

Рассмотрим изменения стека и очереди в каждый момент. Чтобы было понятнее, ниже видео с порядком исполнения вызовов:

{video:https://code.s3.yandex.net/learning-materials/web/screencast_2020-07-03_23-55-20.mp4|autoplay:0}

На стек попадет setTimeout, который запустит Web API планировщика и отправит cb ждать одну секунду. Когда cb отработает, он выведет сообщение в консоль.

После этого на стек вызовов попадёт функция hardWork. Она запускает цикл while, пока не пройдёт три секунды. И после этого выводит в консоль уведомление о завершении.

Но вот что примечательно — cb уже давно попал из Web API в очередь событий, однако, через одну секунду в консоли ничего не появилось. Потому что сначала интерпретатор закончил выполнять то, что лежит на стеке, и только после этого положил на стек колбэк из очереди событий.

Всё потому, что выполнение на стеке всегда в приоритете. Метод setTimeout гарантирует, что событие выполнится не ранее указанного времени.

Но как решить эту проблему?

  1. Разбивать сложную и рутинную работу на более мелкие части, то есть декомпозировать. В таком случае у интерпретатора будет возможность успеть прочитать очередь событий.
  2. Запускать сложные процессы в фоне, чтобы не блокировать главный процесс. Пользователь не хочет получить замороженный интерфейс, пока выполняется какое-то событие (например, при запросе сообщений в чате — интерфейс не блокируется, и можно нажимать на любые кнопки, а поведение выше не позволит делать что-либо). Как запускать процессы в фоне, будет показано в следующих уроках. Для этого можно использовать нулевую задержку либо промисы.

Отмена процесса

Чтобы отменить выполнение setTimeout, в функцию закрытия передают id процесса:

const id = setTimeout(/* ... */);
...
clearTimeout(id);
