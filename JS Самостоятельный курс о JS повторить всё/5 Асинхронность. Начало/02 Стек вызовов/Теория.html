Стек вызовов

Рассмотрим вызов следующей последовательности команд:

(function () {
	function preStart() {
		prepareChatRoom(); // Из предыдущего урока
	}

	function start() {
		preStart();
	}

	start();
})()

Чтобы увидеть порядок вызовов, используют метод console.trace():

function prepareChatRoom() {
	console.log('Подготовить комнату');
  console.trace('prepareChatRoom');
}

(function () {
	function preStart() {
		prepareChatRoom();
	}

	function start() {
		preStart();
	}

	start();
})()

Консоль сообщит примерно следующее:

  • (anonymous) — вызов IIFE,
  • start — вызов функции start,
  • preStart — вызов функции preStart,
  • prepareChatRoom — вызов функции prepareChatRoom.

Все функции попадают в стек в порядке их вызова. Затем они начинают выполняться — уже в обратном порядке, начиная с последней.

prepareChatRoom <——— *вершина стека* 
preStart
start
IIFE

В каждой функции строится собственный стек. В нём описываются:

  • аргументы,
  • локальные переменные,
  • метаданные,
  • указатель на возврат в нужную строчку кода после завершения исполнения функции.

После завершения prepareChatRoom исполняемая программа увидит указатель. Он приведёт её к функции preStart.

preStart <——— *вершина стека*
start
IIFE

Алгоритм продолжит работу до тех пор, пока стек не опустеет. Если стек вызовов пуст, программа завершает работу.

Если в процессе выполнения функции выбросить исключение, стек-трейс будет выглядеть иначе:

function prepareChatRoom() {
  console.log('Подготовить комнату');
}

(function () {
	function preStart() {
    prepareChatRoom(); // Из предыдущего урока
    console.trace('prepareChatRoom');
	}
	
	function start() {
    throw new Error('Some error');
	  preStart();
	}
	
	start();
})()
