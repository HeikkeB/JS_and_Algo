Рубрика «Решаем вместе»: Замыкания

Чтобы приступить к решениям, необходимо разобраться, почему код из badResult не работает.

const badResult = () => {
	for (var i = 0; i < 10; i++) {
	    setTimeout(function() {
	        console.log(i);
	    }, 10);
	}
};

Переменные, объявленные при помощи var, относятся к лексическому окружению функции. Поэтому переменная i существует в единственном экземпляре и мутируется (изменяет своё значение) по ходу выполнения цикла.

Когда цикл полностью отработал — начинается выполнение callback функций из setTimeout. В каждой такой функции есть ссылка на переменную i, которая уже имеет значение 10. Именно поэтому мы и получаем значение 10 в каждом вызове console.log.

Решение через IIFE

const iifeSolution = () => {
    for (var i = 0; i < 10; i++) {
        (function (i) {
            setTimeout(function () {
                console.log(i);
            }, 0);
        })(i)
    }
};

Какие изменения здесь можно увидеть? Всего одно. Самовызывающаяся анонимная функция вида (function(i) { ... })(i). В данном примере мы создаём вокруг setTimeout новое лексическое окружение в виде функции и передаём в него значение переменной i в момент выполнения цикла. Таким образом, значение переменной i в области видимости iifeSolution будет изменяться по ходу выполнения цикла, а внутри анонимной функции останется неизменным. Callback функция из setTimeout сохранит в замыкании теперь не мутирующееся значение i из лексического окружения iifeSolution, а значение i из аргумента анонимной функции.

Решение через let

function es6Solution() {
  for(let i = 0; i < 10; i++) {
    setTimeout(function() {
        console.log(i);
    }, 10);
   }
}

В данном примере изменений ещё меньше. Мы всего лишь изменили способ объявления переменной с var на let. Переменная let существует в блочной области видимости, а значит, остаётся неизменной в рамках каждой итерации нашего цикла. Теперь для каждой из десяти итераций цикла будет существовать своя переменная i с соответствующим значением в области видимости блока цикла.

{
    setTimeout(function() {
        console.log(i);
    }, 10);
}

Решение через bind

const bindSolution = function () {
  for (var i = 0; i < 10; i++) {
     setTimeout(console.log.bind(console, i), 0);
   }
};

Метод bind возвращает функцию с привязанным контекстом и аргументами. Для решения задачи нас в первую очередь интересуют именно аргументы.

function fn(text) {
    console.log(text);
}
const bFn = fn.bind({}, 'Hello');
bFn(); // 'Hello'

Ключевым тут является тот факт, что мы задаём аргументы функции не в момент вызова (как обычно), а в момент «биндинга». Таким образом, мы легко можем указать аргумент в момент каждой итерации цикла с нужным значением.
