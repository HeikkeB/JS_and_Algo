Как работает setInterval

Метод setInterval похож на setTimeout, но у него есть важное отличие. Он будет выполнять запланированное действие регулярно, через заданный промежуток времени, в то время как setTimeout сработает только один раз. Примеры использования setInterval:

setInterval(() => {
	console.log('Я буду выполняться через каждые 2 секунды')
}, 2000);

// Или так
setInterval(console.log, 2000, 'Я буду выполняться через каждые 2 секунды');

Подробнее про сигнатуру можно почитать на MDN.

При простом случае работа со стеком и очередью событий не отличается от setTimeout. Единственное исключение: процесс идёт бесконечно, пока пользователь сам его не закроет.

Ожидаемый таймлайн для setInterval:

Рассмотрим ту же проблему выполнения кода в фоне, что и в случае setTimeout.

Сделаем так, чтобы интервал запускал внутри себя функцию, которая работает ~50мс:

function hardWork() {
	// этот код выполняется 50 мс
}
console.log('Start');

setInterval(function cb() {
	hardWork();
	console.log('Done');
}, 100);

Ожидается, что через каждые 100мс будет запускаться cb. Однако, если нарисовать таймлайн того, как это работает в реальности, можно прийти к следующему выводу: setInterval гарантирует, что событие отработает не ранее указанного времени.

На первых трёх запусках можно увидеть, что hardWork отработал через разное время (51ms, 53ms, 52ms). То есть уже нет гарантий, что сдвиги будут одинаковые и через одно и то же время будет перезапуск.

Реальная задержка между вызовами вышла меньше, чем указано в коде. Подобное поведение валидное, потому что время, которое затрачивается на выполнение колбэка, использует часть заданного интервала времени.

Метод setInterval менее гибкий, чем запуск setTimeout внутри setTimeout, о чём поговорим в следующем уроке.

Закрытие процесса

Чтобы остановить процесс выполнения setInterval, в функцию закрытия передают id процесса:

const id = setInterval(/* ... */);
...
clearInterval(id);
