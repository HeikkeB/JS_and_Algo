Ответы на задания

1. Что будет, если попытаться добавить новый элемент в массив const arr = ['чашка', 'чайник', 'табурет']?

  • (-) Ошибка. Переменная объявлена при помощи const, а значит менять значение нельзя

  • (+) Элемент добавится в массив

    • Если переменная объявлена через const, ей нельзя присвоить новое значение, но если в ней хранится, например, объект, то его свойства можно изменить.

  • (-) Ошибки не будет, но массив не изменится

2. На что будет указывать this внутри тела стрелочной функции?

  • (-) По-разному. Всё зависит от того, как именно её вызовут

  • (-) Ни на что. Он всегда будет равен undefined

  • (+) На окружающий её контекст

    • Потому что у стрелочной функции нет this.

3. Что будет в Set?

const set = new Set(['ночь', 'улица', 'улица', 'фонарь']);

console.log(set)

  • (-) jsx ['ночь', 'улица', 'улица', 'фонарь']

  • (-) jsx { 0: 'ночь', 1: 'улица', 2: 'улица', 3: 'фонарь' }

9. Что будет выведено в консоль?

class Person {
	constructor(name) {
    this.name = name;
  }

  static changeName(newName) {
    this.name = newName;
    return this.name;
  }
}

const sonya = new Person({ name: 'Соня' });
console.log(sonya.changeName('Машенька'));

  • (-) 'Соня'

  • (-) 'Машенька'

  • (+) TypeError

    • Метод changeName существует только в классе Person, но его нет в экземпляре, который хранится в переменной sonya, потому что этот метод статический.

11. Что будет выведено в консоль?

function sum(a, b = a) {
  return a + b;
}

  • (+) 6

    • Мы задали параметр по умолчанию, то есть b = a. Если в функцию передать только один аргумент, то второй аргумент будет такой же, а значит нашем случае 3 + 3 = 6.

  • (-) NaN

  • (-) undefined

  • (-) ReferenceError

12. Что будет выведено в консоль?

const add = () => {
  const cache = {};
  return num => {
    if (num in cache) {
      return `Из кеша: ${cache[num]}`;
    } else {
      const result = num + 10;
      cache[num] = result;
      return `Посчитано: ${result}`;
    }
  };
};

const addFunc = add();
console.log(addFunc(10));
console.log(addFunc(10));
console.log(addFunc(5 * 2));

  • (-) 'Из кеша: 20', 'Из кеша: 20', 'Из кеша: 20'

  • (+) 'Посчитано: 20', 'Из кеша: 20', 'Из кеша: 20'

    • Чудеса мемоизации и замыкания. Все три раза мы вызвали функцию addFunc с тем же аргументом. В первый раз аргумент и результат сохранились в объект cache. При следующих вызовах, поскольку аргумент уже является ключем в объекте cache, результат вернулся оттуда.

  • (-) 'Посчитано: 20', 'Из кеша: 20', 'Посчитано: 20'

13. Что такое ООП?

  • (+) Это методология программирования, основанная на представлении программы в виде совокупности объектов, каждый из которых является экземпляром определенного класса, а классы образуют иерархию наследования.

  • (-) Это парадигма декларативного программирования, в которой программы создаются путем последовательного применения функций, а не инструкций.

  • (-) Это парадигма программирования, в которой задаётся последовательность действий, необходимых для получения результата.

14. Какие основные принципы ООП?

  • (-) Связанность, наследование и иденпотентность

  • (+) Абстрагирование, инкапсуляция, наследование и полиморфизм

  • (-) Больше, выше, сильнее

15. Что такое «Фабрика»?

  • (+) Функция или метод для создания разных объектов с одинаковым интерфейсом

  • (-) Класс, который создаёт одинаковые функции с разными параметрами

  • (-) Функция, которая при каждом вызове добавляет новый элемент в массив или объект
