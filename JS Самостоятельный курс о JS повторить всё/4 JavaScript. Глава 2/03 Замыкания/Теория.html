Замыкания

К сожалению, не многие могут правильно своими словами рассказывать, что такое «замыкание» в JavaScript. Достаточно уметь показать, как оно работает, и понимать принципы, о чём и пойдёт речь в этом уроке.

LexicalEnvironment

У каждой функции или блока кода есть скрытый объект, который содержит локальные переменные, ссылку на лексическое окружение внешней функции или блока кода и другую информацию. Объект лексического окружения состоит из двух частей:

  • Environment record — это объект, в котором содержатся локальные переменные и значение this как свойства,
  • ссылки на внешнее лексическое окружение.
function findChats(title) {
	// LexicalEnvironment = {title: 'Мой суперчат', result: undefined}
	const result = chats.filter(...);
	// LexicalEnvironment = {title: 'Мой суперчат', result: [...]}
	return result;
} 
 
findChats('Мой суперчат'); 
// LexicalEnvironment = {title: 'Мой суперчат', result: [...]}

Замыкание — это способность функции «запоминать» то лексическое окружение, в котором она была объявлена

Замыкания «на пальцах»

Замыкания — действительно сложная тема в мире JavaScript, а сложные темы лучше разбирать на пальцах. Представьте, что вы пишете функцию для работы с HTTP запросами:

function httpRequest(method, url) {
	// Реализация функции
}

Аргументы функции:

  • method — HTTP метод запроса (например, get, post, put или delete),
  • url — адрес сервера.

Отлично! Теперь можно вызвать функцию:

httpRequest('get', 'http://example.com');

Это будет работать, но кажется, что не очень удобно каждый раз указывать метод. Перепишем httpRequest так, чтобы была возможность указать метод единожды, а дальше указывать только адрес:

function httpRequestBuilder(method) {
	return (url) => {
		// Реализация функции
	}
}

const getHttpRequest = httpRequestBuilder('get');

getHttpRequest('http://example.com');
getHttpRequest('http://example2.com');
getHttpRequest('http://example3.com');

Разберёмся, что здесь происходит. Теперь у вас есть функция httpRequestBuilder, которая принимает на вход метод запроса и возвращает другую функцию. После вызова httpRequestBuilder вы получаете ту самую возвращаемую функцию в переменной getHttpRequest. Обратите внимание, функция httpRequestBuilder уже отработала. Она и соответствующее ей лексическое окружение (со всеми переменными) должны удалиться из памяти, но у вас есть доступ к method из той самой возвращаемой функции:

return (url) => {
	// Реализация функции
}

В этом и заключается вся магия замыкания. Теперь лексическое окружение функции httpRequestBuilder не будет очищено из памяти до тех пор, пока существует функция getHttpRequest. Вы создали замыкание. Возвращаемая функция была создана в момент исполнения функции httpRequestBuilder и «запомнила» внешнее лексическое окружение.

Внешние переменные

Функция обращается не только к внутренним переменным, но и к внешним. Процитируем «Современный учебник JavaScript»:

Интерпретатор при доступе к переменной сначала пытается найти её в текущем LexicalEnvironment, а затем, если не нашёл, — во внешнем лексическом окружении. В данном случае им является window.

Рассмотрим работу внешних переменных.

Напишите функцию canGetCount. Она принимает число N и возвращает функцию, которая:

  • первые N вызовов возвращает строку “yes”,
  • каждый следующий вызов возвращает строку “no”.

Решение может выглядеть так:

function canGetCount(n) {
  let counter = n;
  
  return () => {
    if (counter > 0) {
      counter--;
      return 'yes';
    } else {
      return 'no';
    }
  }
}

const getOne = canGetCount(2);

getOne()  // yes
getOne()  // yes
getOne()  // no

Рассмотрим функцию с точки зрения лексического окружения и замыкания:

const someOtherVar = 'string for test';

function canGetCount(n) { // далее мы рассмотрим LexicalEnvironment при вызове canGetCount(2)
  let counter = n;

	function getOne() {
		const someMagicNumber = 42;
	
	  if (counter > 0) {
	    counter--;
	    return 'yes';
	  } else {
	    return 'no';
	  }
		// LE = {someMagicNumber: 42}
  };
  // LE = {someOtherVar: 'string for test', n: 2, counter: 2, getOne: func}
  return getOne;
}

После создания функции, будет создано скрытое свойство [[Scope]], которое ссылается на её лексическое окружение.

Если рассматривать задачу про счётчик, то в самом начале у функции canGetCount окружением будет объект window.

Поскольку переменной someOtherVar в canGetCount не существует, в начале будет происходить поиск внутри самой функции, затем поиск переменной во внешнем окружении (в примере оно совпадает с глобальным) и так далее до самого верхнего уровня. Таким образом, someOtherVar попадёт в лексическое окружение самой функции canGetCount.

Эта задача работает благодаря замыканию. После вызова функции canGetCount, в замыкание getOne попадает ссылка на counter — и значение этой переменной меняется.

Сборщик мусора не удалит getOne, поскольку на неё есть как минимум одна ссылка, а значит, доступ к ней будет. Подробнее работу сборщика мусора рассмотрим в следующих спринтах.

Ещё пример замыкания:

function sum(a) {
  return function(b) {
    return a + b;
  }
}
 
// const sum = a => b => a + b
 
const plus = sum(3);
plus(6); // 9

В замыкание попадёт первый операнд a, далее к нему прибавляем второй операнд b, который передаётся через функцию plus.

Рассмотрим лексическое окружение функции sum:

function sum(a) { // 3
  return function(b) { // 6
    return a + b;
		// LE = {b: 6}
		// .[[Scope]] = {a: 3}
  }
} // LE = {a: 3}
// sum.[[Scope]] = window

Порядок инициализации

В JavaScript большую роль играет порядок инициализации переменных и функций. Из-за этого можно столкнуться с ошибками вроде «Почему одни функции я могу вызывать до объявления, а другие при вызове выбрасывают ошибки?».

Function Declaration попадёт в лексическое окружение не в момент вызова или когда выполнение дойдёт до этой функции, а в самом начале. Другие же виды объявления функций через присвоение переменной будут определены только в тот момент, когда к ним дойдёт выполнение.

Именно поэтому уже в самом начале функция logger попадёт в лексическое окружение:

// LE = {logger: func, func1: undefined, func2: undefined}
// func1 и func2 ещё не определены и при вызове до объявления будет выброшено исключение
logger(); // logger
func1(); // ReferenceError: Cannot access 'func1' before initialization
func2(); // Сюда не дойдёт из-за ошибки в func1
				 // Но если бы дошло, получили бы такую же ошибку
				 // ReferenceError: Cannot access 'func2' before initialization

function logger() {
  console.log('logger');
}

const func1 = () => {
  console.log('func1');
};

const func2 = function() {
  console.log('func2');
};

Итог

Концепция замыкания в языке одна из самых сложных для понимания. Даже опытные программисты не всегда могут сходу объяснять, как оно работает. Возможно, вам потребуется больше времени, чтобы разобраться с ним, но это нормально и вы уже сделали первый шаг.

Понимание замыкания пригодится в решении задач и на собеседованиях.

Если кратко, то замыкание — это лексическое окружение функции и принципы работы языка с этим лексическим окружением. Немного неточно называть замыкание «функцией» с внешними переменными.
