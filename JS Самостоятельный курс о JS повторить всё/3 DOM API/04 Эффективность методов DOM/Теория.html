Эффективность методов DOM

Рассмотрим пример вставки элементов в DOM:

for (let i = 0; i < 1000; i++){
  const element = document.createElement('div');
  element.textContent = `${i} text`;
  document.body.appendChild(element);
}

Или же изменим значение innerHTML:

for (let i = 0; i < 1000; i++){
  document.getElementById(`${i}`).innerHTML = `${i} другой text`;
}

Из общего у этих листингов кода можно выделить одно — они заставляют браузер и компьютер пользователя «кипеть». Если попытаться то же самое сделать сразу со множеством элементов, интерфейс полностью утратит отзывчивость. Пользоваться сайтом станет невозможно.

В этом уроке разберёмся, почему работа с DOM так затратна для ресурсов браузера.

Почему это дорого?

DOM-дерево — сложная структура данных с множеством процессов и обработчиков. Каждый раз залезать в него — очень медленная операция. А если делать это в цикле с большим количеством элементов, можно получить критическую ошибку:

Подобные ошибки могут появляться из-за утечек памяти и некорректного использования JavaScript, потому стоит избегать частой работы напрямую с DOM.

Оптимизации

Уменьшить число вызовов записи к DOM

Можно делать обновления небольшими порциями. Например, при вставке использовать фрагменты, которые добавляются в структуру данных JavaScript, и только после завершения цикла — вставить их в DOM:

const fragment = document.createDocumentFragment();
for(let i = 0; i < 1000; i++){
  const element = document.createElement('div');
  fragment.appendChild(element);
}
document.body.appendChild(fragment);

Не перестраивать постоянно DOM

Удаление узлов из DOM-дерева — тоже тяжеловесная операция. Если постоянно комбинировать её со вставкой, этого будет достаточно, чтобы браузер начал тормозить.

Подобные вещи решаются через CSS. Компонентам понадобится класс hidden, который будет их скрывать:

.button__visible-hidden {
	display: none;
}

Такой класс можно переключать с помощью DOM API и метода toggle.

О других способах скрывать элементы — в статье на Хабре.

Мемоизация

Данный инструмент представляет собой механизм кеширования.

Представьте функцию, которая принимает аргументы и на их основе выполняет сложные вычисления. Например, перебор пароля от Wi-Fi точки соседа. Алгоритм детерминированный — при одних и тех же аргументах он выдаёт один и тот же ответ.

Алгоритм работал очень долго, но потом функция снова была вызвана с теми же аргументами. Зачем заново проводить тяжёлые вычисления, если можно вернуть результат, который посчитан до этого?

const result = someHardFunction(1, 2); // Супердолгие вычисления 
const result1 = someHardFunction(3, 4); // Супердолгие вычисления
const result3 = someHardFunction(1, 2); // Вычислений вообще не будет
										// Ответ быстро вернётся из кеша (из хеш-мапы или объекта в JS)
