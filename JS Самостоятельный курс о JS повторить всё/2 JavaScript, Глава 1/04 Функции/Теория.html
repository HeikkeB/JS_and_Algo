Функции

Функция — блок кода, который позволяет переиспользовать существующий код. Может иметь входные аргументы и возвращать значение. Также это «вызываемый» объект, то есть объект с внутренним свойством [[Call]], которое позволяет его вызвать.

Если функция принимает в виде аргументов другие функции или возвращает их, то её называют функцией высшего порядка.

Объявление функций (Function Declaration)

function greet(name) {
  return `Hello ${name}`;
}

greet('Sergey'); // Hello Sergey

//////////////

greet('Sergey'); // Hello Sergey

function greet(name) {
  return `Hello ${name}`;
}

Обычное объявление. Такую функцию можно вызвать до объявления — благодаря эффекту всплытия (Hoisting).

Вероятно, вы уже слышали про всплытие в JavaScript. Важно понимать порядок инициализации и помнить отличия использования до ES6 и после. Подобные возможности языка по незнанию превращаются в потенциальные ошибки. Отличные статьи про Hoisting в целом и если речь идёт о переменных.

Функциональные выражения (Function Expression)

const greet = function (name) {
  return `Hello ${name}`;
};

greet('Sergey'); // Hello Sergey

Функция будет присвоена переменной и описана без имени. Такую функцию нельзя вызвать до объявления.

Именованные функциональные выражения (Named Function Expression)

const count = function step(number) {
  if (number > 0) {
    step(number - 1);
  }
};

count(5);

Имя функции доступно только изнутри, имя переменной доступно везде.

Стрелочные функции (Arrow function)

const greet = (name) => {
  return `Hello ${name}`;
};

const greet = (name) => `Hello ${name}`;    

const greet = name => `Hello ${name}`;

Такая сокращённая запись функций подходит для небольшого блока кода, которому не нужен собственный контекст.

Существенное отличие — это контекст. У стрелочных функций нет своего this. Он берётся из верхнего окружения. Рассмотрим пример:

const test = {
	name: 'we want it',
	makeAnonFunction: function() {
	  return function() {
	    console.log(this.name);
	  };
	},
	
	makeArrowFunction: function() {
	  return () => {
	    console.log(this.name);
	  };
	}
};

При вызове anon функции, получим undefined в консоли, так как у анонимной функции внутри makeAnonFunction нет свойства name.

Но при вызове стрелочной функции получим в консоли ожидаемую строку — 'we want it':

> const anon = test.makeAnonFunction();
> const arrow = test.makeArrowFunction();

> anon();
undefined

> arrow();
we want it

Аргументы функций

В функции аргументы можно передать по ссылке и по значению.

const message = {
  user: {
    messages: 4210
  }
};
        
function incrementCount(count) {
  count++;
}
        
incrementCount(message.user.messages);
message.user.messages; // 4210

//////////////////

function incrementCount2(user) {
  user.messages++;
}
        
incrementCount2(message.user);
message.user.messages; // 4211

В первом случае передаётся значение messages, к которому прибавляется единица. Поэтому после выполнения incrementCount в message.user.messages будет лежать старое значение. Но если передать всю ссылку на объект, как во втором случае, то после выполнения incrementCount2 можно увидеть ожидаемое увеличенное значение.

Функции умеют принимать как заданное число аргументов, так и неопределённое.

function sumNumbers() {
  let sum = 0;

  for (let i = 0; i < arguments.length; i++) {
    sum += arguments[i];
  }

  return sum;
}

sumNumbers(1, 2, 3); // 6

//////////////////////////////////

function sumNumbers() {
  const args = Array.from(arguments); // Это старый способ получения аргументов

  let sum = 0;

  args.forEach(function (num) {
    sum += num;
  });

  return sum;
}

sumNumbers(1, 2, 3); // 6

//////////////////////////////////

function sumNumbers(...args) { // Это способ из ES6+
  return args.reduce((sum, num) => {
    return sum + num;
  }, 0);
}

sumNumbers(1, 2, 3); // 6

В функциях можно описывать значения по умолчанию. До ES6 часто можно было встретить такой код:

function some(a, b) {
	a = a || 0;
	b = b || function() {};
}

Чем он плох? Неочевидная проблема, которая заставляет подумать про типы: оператор || работает с "", null, NaN, false, 0 иначе, чем можно было бы предположить.

let a = "";
a = a || function() {} // function() {}. Значение "" не сохранилось

Поэтому подобный стиль работы со значениями по умолчанию считают скорее антипаттерном.

Хорошим решением до ES6 было явное сравнение с undefined:

function some(a) {
	a = typeof a === "undefined" ? "some value" : a;
}

Громоздко и неудобно.

Спасибо ES6, теперь есть нормальный способ объявить значения аргументов по умолчанию:

function some(a = function() {}) {
	console.log(a);
}

some(''); // Теперь мы получим пустую строку в консоли

Сами функции можно передать в качестве аргументов. Именно это свойство делает их объектами высшего порядка. На самом деле вы уже пользуетесь ими. Например, методы массивов (forEach, map, reduce и другие) — это функции высшего порядка, поскольку в качестве аргумента они принимают другие функции.

/*
message = {
  id: '228',
  text: 'Привет, Вася.\r\nКак дела?',
  user: {
      id: 42,
      name: 'Николай',
      login: 'nikolashka',
			[someKey]: 'Петров',
  },
  'start-year': 2007,
	tags: ['javascript', 'html', 'css'],
};
*/

let result = [];
// Выбираем только пользователей с тегами "javascript"
function filterMessagesByTag(message) {
	const {tags} = message;

  if (Array.isArray(tags) && tags.includes('javascript')) {
      result.push(message);
  }
}

// Теперь в result лежат отфильтрованные твиты
messages.forEach(filterMessagesByTag);

Преобразования можно выделять в целые цепочки вызовов. Например, мы хотим сделать список из сообщений:

let result = messages
  .filter(filterMessagesByTag)
  .map(getHTML)
  .join('\n');
        
// Выбираем только пользователей с тегами "javascript"
function filterMessagesByTag(message) {
	const {tags} = message;

  return Array.isArray(tags) && tags.includes('javascript');
}
        
// Превращаем массив объектов твитов в HTML-строки
function getHTML(message) {
	return `<li>
	    <div>${message.user.name}</div>
	    <div>${message.text}</div>
	    <div>${message.tags.join(',')}</div>
	  </li>`;
}

// Теперь в result лежит HTML c деревом сообщений
result = `<ul>${result}</ul>`;

Особенности функций

  1. Может возвращать или не возвращать значение (будет неявно возвращать undefined):

    function noop() {
}


function noop() {
	return undefined;
}


function noop() {
	return;
}


console.log(noop()); // undefined

  2. Являются объектами высшего порядка.

  3. Может быть переданы в другие функции в качестве аргумента:

    // Принимаем функцию в функции как аргумент
function modifyItems(items, modify) {
	return items.map(modify); // то же самое можно записать иначе: .map(item => modify(item))
}
const numbers = [1, 2, 3, 4];


// Пример с использованием стрелочной функции
const multiplyBy2 = item => item * 2;
const doubleValues = modifyItems(numbers, multiplyBy2);
// [2, 4, 6, 8]

Здесь функция modifyItems в качестве аргументов получает массив элементов и функцию-модификатор modify. Затем modifyItems применяет функцию modify к каждому элементу массива.

Такой функции мы можем передать любую функцию, которая работает с элементами массива. И передать её как items.map(modify) вместо map(item => modify(item)). Кода меньше, читается проще.

Каррирование

Каррирование — продвинутая техника для работы с функциями. Она используется не только в JavaScript, но и в других языках. По сути это трансформация функций таким образом, чтобы они принимали аргументы не как f(a, b, c), а f(a)(b)(c).

function getRaiser(pow) {
  return function (num) {
    return num ** pow;
  }
}

const squared = getRaiser(2);
squared(2); // 4
squared(3); // 9

const cube = getRaiser(3);
cube(2); // 8
cube(3); // 27

// Или вот так

getRaiser(2)(3); // 9
getRaiser(3)(2); // 8
