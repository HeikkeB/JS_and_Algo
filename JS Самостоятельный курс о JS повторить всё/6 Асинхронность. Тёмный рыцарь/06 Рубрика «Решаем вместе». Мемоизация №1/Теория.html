Рубрика «Решаем вместе»: Мемоизация №1

Мемоизация — это особая форма кеширования, которая включает в себя кеширование возвращаемого значения функции на основе её параметров. Если мемоизированная функция уже вызывалась с этими же аргументами, она должна сразу вернуть результат.

Напишем функцию memoize, которая принимает на вход другую функцию (далее будем называть её «оригинальная функция») и создаёт новую мемоизированную версию оригинальной функции. Стоит обратить внимание на то, что memoize должна уметь мемоизировать в том числе и асинхронные функции.

Также есть дополнительное условие про время жизни кеша. Этот пункт вынесем в отдельный шаг, и сначала реализуем memoize, которая будет мемоизировать функции на бесконечный период времени.

Вариант без времени жизни кеша и асинхронной работы

Вызовы функции можно хранить в кеше, использовав обычный объект в качестве хранилища. Ранее уже отмечалось, что мемоизация зависит от переданных аргументов, поэтому аргументы являются ключом, а результат выполнения — значением:

function sum(a, b) {
  return a + b;
}

const memoizedSum = memoize(sum);

memoizedSum(4, 5); // 9
memoizedSum(3, 3); // 6
  
// Псевдокод для хранилища:
// cache = {
//   (args: 4, 5): 9
//   (args: 3, 3): 6
// };

Есть две проблемы: где хранить объект с кешем и как представить аргументы в качестве ключа, так как ключом объекта может быть только строка или символ.

Первая проблема решается в JavaScript довольно просто. Используем замыкание:

function memoize(fn) {
  const cache = {};
  
  return function (...args) {
    // ...
  };
}

Решение второй проблемы уже написано выше — использовать в качестве ключа строку, сформированную из аргументов. Это можно сделать с помощью вспомогательной функции, которая будет генерировать из аргументов строковый ключ:

function generateKey(args) { 
  return args.map(arg => `${typeof(arg)}<${String(arg)}>`).join(',');
}

На примере выше результат будет следующим:

generateKey([4, 5]); // "number<4>,number<5>"
generateKey([3, "foo"]); // "number<3>,string<foo>"

Важно сохранять тип переменной, чтобы значения вроде "0" и 0 или "true" и true воспринимались как разные. Эта функция не идеальна, например, для объектов, ведь она будет выводить значение как [object Object]. Можете доработать её самостоятельно, рекурсивно обходя массивы и объекты.

generateKey([{ foo: "bar" }]); // "object<[object Object]>"

И снова к memoize. Она вернёт новую функцию, при вызове которой, если ключ аргументов есть в кеше, значение вернётся из него. Если нет, то выполнится оригинальная функция, результат сохранится в кеше и только после этого вернётся:

function generateKey(args) { 
  return args.map(arg => `${typeof(arg)}<${String(arg)}>`).join(',');
}

function memoize(fn) {
  const cache = {};
  
  return function (...args) { 
    const key = generateKey(args);
    let result = cache[key];
    
    console.log(typeof result === 'undefined' ? 'Execute fn' : 'From cache');
    
    if (typeof result === 'undefined') {
      result = fn(...args);
      
      cache[key] = result;
    }
    
    return result;
  };
}

Проверим функцию в действии:

function sum(a, b) {
  return a + b;
}

const memoizedSum = memoize(sum);

memoizedSum(4, 3); // Execute fn(), 7
memoizedSum(4, 3); // From cache, 7
memoizedSum(4, 4); // Execute fn(), 8

Версия с поддержкой асинхронных функций

При работе с асинхронными функциями можно использовать Promise.resolve(). Метод даёт возможность строить цепочки вызова промисов. Когда берём значение из кеша или вызываем функцию, оборачиваем их в Promise.resolve:

function memoize(fn) {
  const cache = {};
  
  return function (...args) { 
    // ...
	  
    if (typeof result === 'undefined') {
      return Promise.resolve(fn(...args)).then(result => {
        cache[key] = result;
        return result;
      });
    }
    
    return Promise.resolve(result);
  };
}

Проверим метод в работе:

function getData() {
  return new Promise(resolve => setTimeout(() => resolve(42), 1000));
}

const memoizedGetData = memoize(getData);

memoizedGetData()
  .then(result => console.log(result)) // получаем долго
  .then(() => memoizedGetData())
  .then(result => console.log(result)) // получаем быстро, из кеша
  .then(() => memoizedGetData())
  .then(result => console.log(result)); // получаем быстро, из кеша

Функция вызывается без параметров. В таком случае ключ в кеше будет пустой строкой, так как generateKey([]) вернёт именно пустую строчку.

Версия со временем жизни кеша

Чтобы определить, актуально ли значение в кеше, можно сравнить текущее время (которое можно получить с помощью Date.now()) со временем создания записи в кеше, просуммированным с переданным временем жизни кеша во втором аргументе. Если текущее время меньше, то берём значение из кеша, иначе запрашиваем его заново из оригинальной функции.

Для каждого набора аргументов время актуальности в кеше должно быть своё, поэтому по ключу будет возвращаться не результат, а объект с результатом и временем, до которого запись актуальна:

function memoize(fn, timeout) {
  const cache = {};
  
  return function (...args) { 
    const key = generateKey(args);
    const result = cache[key];    
	  
    if (typeof result === 'undefined' || Date.now() > result.expire) {
      console.log('Execute fn');
      return Promise.resolve(fn(...args)).then(value => {
        cache[key] = { value, expire: Date.now() + timeout };
        return value;
      });
    }

    console.log('From cache');
    
    return Promise.resolve(result.value);
  };
}

Проверим функцию в действии:

function getData() {
  return new Promise(resolve => setTimeout(() => resolve(42), 1000));
}

const memoizedGetData = memoize(getData, 1000);

memoizedGetData()
  .then(result => console.log(result)) // получаем долго
  .then(() => memoizedGetData())
  .then(result => console.log(result)) // получаем быстро, из кеша
  .then(() => memoizedGetData())
  .then(result => console.log(result)) // получаем быстро, из кеша
  .then(() => {
    // получаем долго, считается заново
    setTimeout(() => memoizedGetData().then(result => console.log(result)), 5000);
  });

Финальный вариант:

function generateKey(args) { 
  return args.map(arg => `${typeof(arg)}<${String(arg)}>`).join(',');
}

function memoize(fn, timeout) {
  const cache = {};
  
  return function (...args) { 
    const key = generateKey(args);
    const result = cache[key];    
	  
    if (typeof result === 'undefined' || Date.now() > result.expire) {
      return Promise.resolve(fn(...args)).then(value => {
        cache[key] = { value, expire: Date.now() + timeout };
        return value;
      });
    }
    
    return Promise.resolve(result.value);
  };
}
